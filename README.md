# heybox Lite

heybox Lite 是一个为 Android 手表做的小黑盒社区第三方客户端。

它不是想把手机端完整塞进手表里，而是尽量把“刷社区、看帖子、看图、看评论、做一些常用互动”这些事变得能在小屏幕上用。这个项目最早参考了 HeyWear 的方向，后来围绕方屏、圆屏和低性能设备做了很多自己的取舍：少一点装饰，多一点可读内容；能滑动解决的地方，就尽量少塞按钮。

当前正式版：`2.0.4`

2.0 线主要面向 Android 7.0 及以上设备测试。构建配置仍保留更低的 `minSdk`，是为了让部分旧手表可以尝试安装，但实际体验和兼容性以 Android 7.0+ 为准。

## 现在能做什么

- 扫码登录小黑盒账号，也可以在游客状态下浏览部分内容。
- 浏览社区信息流，进入帖子详情，查看正文、图片和评论。
- 帖子正文支持官方表情、图片缓存、图片预览和缩放查看。
- 支持点赞、收藏、关注、评论等常用互动；这些接口依赖小黑盒当前风控策略，偶尔可能失效。
- 支持我的收藏、浏览历史、个人动态空间。
- 支持搜索帖子，并保留搜索结果返回位置。
- 支持启动公告、更新检查和应用内下载更新。
- 支持日志导出到 `Download/heyboxlite`，方便手表上排查问题。
- 支持主题色、字号、行距、边距、圆屏适配、滑动返回等显示设置。
- 支持底部悬浮胶囊导航，主页/我的之间可以横向切换。
- 后台服务支持公告、更新信息和简单在线人数统计。

## 2.0 做了哪些变化

2.0 主要是一次体验重做，不只是改版本号。

- 首页和帖子详情改成更克制的黑白灰风格，减少 1.x 那种临时拼出来的感觉。
- 底部导航改成悬浮胶囊，不再占用正文布局空间。
- 搜索入口移到主页顶部，底部只保留核心页面切换。
- 设置页、二级菜单、帖子页补上更顺手的滑动返回。
- 图片查看器支持缓存预览、双击分级放大、点击空白退出。
- 帖子正文图片支持渐进显示，正文里不再一直转圈等整张图加载完。
- 主页图片加载失败时显示灰色占位，不用上一张旧图硬顶上来。
- 评论区和发评论面板重新整理了样式，尽量少挡内容。
- 新增“记住帖子阅读位置”开关，主页和搜索列表返回位置不受这个开关影响。
- 日志导出、缓存清理、交流群二维码、公告列表等小功能做了文案和交互整理。

## 还不完美的地方

这个项目依赖小黑盒非公开接口，所以有些东西不是“写完就永远稳定”：

- 签到功能仍属于实验状态。小黑盒移动端有额外签名和风控，当前实现不保证每天都能成功。
- 点赞、收藏、关注、评论这些写入操作可能因为账号状态、风控、接口参数变化而失败。
- 官方表情、帖子正文结构、图片字段会随接口变化，后续可能还需要继续修。
- 低版本 Android 和不同手表 ROM 的 Web/TLS/安装策略差异很大，2.0 优先保证 Android 7.0+。

如果只是想稳定刷社区，建议先把签到当成可选功能，不要把它当作主流程的一部分。

## 构建

需要：

- JDK 17
- Android SDK
- Gradle Wrapper 已包含在仓库中

调试构建：

```powershell
.\gradlew.bat assembleDebug
```

正式构建：

```powershell
.\gradlew.bat assembleRelease
```

Release 包会复制到：

```text
dist/heybox-Lite-版本号.apk
dist/heybox-Lite-latest.apk
```

如果本地没有 release 签名配置，2.0 preview 包名会使用 debug 签名构建，方便测试安装。正式发布时请配置自己的签名，不要把 keystore、密码、Cookie、HAR、抓包文件或日志提交到仓库。

支持的签名配置方式：

- `release-signing.properties`
- 环境变量 `HEYBOX_RELEASE_STORE_FILE`
- 环境变量 `HEYBOX_RELEASE_STORE_PASSWORD`
- 环境变量 `HEYBOX_RELEASE_KEY_ALIAS`
- 环境变量 `HEYBOX_RELEASE_KEY_PASSWORD`

## 项目结构

```text
app/src/main/java/com/openzen/heyboxcommunity/
```

主要代码还在 Java View 体系里，没有切到 Compose。这样做不是最时髦，但对旧手表、小屏设备和安装包体积更友好。

几个比较重要的模块：

- `MainActivity`：页面组织、列表、详情、设置、评论等主要界面。
- `ApiClient`：小黑盒接口请求。
- `SessionStore`：本地设置、登录态和开关。
- `ImageLoader`：图片缓存、缩略图、正文图和查看器加载。
- `UpdateChecker` / `AnnouncementChecker`：更新与公告。
- `SignInManager`：签到实验功能，和其它互动逻辑隔离。
- `PresenceReporter`：后台在线人数心跳。

代码里仍有一些历史包袱，尤其是早期为了兼容接口和手表设备堆出来的逻辑。后续会继续拆，但会优先保证“能用”和“别把现有功能改炸”。

## 后台服务

App 的更新、公告和在线人数统计依赖自建服务器接口。后台只做很轻量的事情：

- 返回当前版本号、更新日志和 APK 地址。
- 返回公告列表。
- 接收 App 前台心跳，按 IP 统计当前在线和今日独立访问。

在线人数按 IP 去重，不是账号级统计，也不用于识别个人。

## 安全与隐私

- 登录态只保存在本机。
- 日志导出用于排查问题，里面可能包含接口错误、设备环境和部分请求状态。
- 不要公开上传自己的 Cookie、HAR、二维码登录结果、抓包文件或诊断日志。
- Release 构建会开启混淆和资源压缩。
- 项目会尽量避免明文流量，但第三方接口和服务器配置仍需要持续检查。
