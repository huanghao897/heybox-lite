# heybox Lite

面向方屏 Android 手表的小黑盒社区第三方客户端，重点适配 CD20 Max 一类小尺寸方屏设备。支持 Android 4.0（API 14）及以上系统。

## 功能

- 二维码登录与登录态加密保存
- 社区信息流、帖子详情、图片与评论
- 二级评论分批展开和收起
- 图片中心缩放全屏预览、双指缩放与原图查看
- 个人资料、收藏夹(目前不可用)与浏览历史
- 首页下拉刷新、社区帖子搜索与历史记录搜索
- 搜索页保留最近搜索记录并支持一键清空
- 日间/夜间主题、十二组主/副色预设与缓存清理
- 界面、文字、边距及帖子正文字体/间距实时预览
- 独立界面预览页，模拟帖子、正文和评论层级
- 可配置开屏动画、开屏文字和显示时长
- 可关闭的启动自动更新检查
- 一键恢复默认显示设置
- 首页文章类型标识
- 关于页、版本信息和 GitHub Release 更新检查
- 针对小尺寸方屏的紧凑布局

## 构建

需要 Java 17 和 Android SDK。项目当前使用 Android API 36、AGP 8.13.2 和 Gradle 8.13。

```powershell
.\gradlew.bat assembleDebug
```

正式版本需要在本地配置签名信息后构建：

```powershell
.\gradlew.bat assembleRelease
```

签名文件、`release-signing.properties`、HAR、Cookie、构建产物和 R8
`mapping.txt` 均被排除在版本控制之外。

## 安全说明

- Android 6.0 及以上使用 Android Keystore + AES-GCM 保存 Cookie。
- Android 4.0–5.1 使用设备与应用签名绑定的 AES-CBC + HMAC 加密保存 Cookie，登录状态可跨重启保留；受旧系统能力限制，安全强度低于 Android 6.0 及以上。
- 网络请求头、接口路径和鉴权字段统一封装；release 开启 R8 full mode 与资源压缩。
- release 版本关闭调试、禁止明文网络流量与备份，并包含轻量签名完整性检查。

这些措施用于提高基础安全性，不能替代服务端鉴权和风控。
